## 2. 产品目标

### 2.1 核心目标

帮助用户在设定的学习时段内只访问白名单网站，减少分心行为，并在会话结束后查看专注数据汇总。

### 2.2 产品成功标准

1. 用户可以在填写学习目的并选择时长后快速进入专注状态。
2. 非白名单网站在专注期间会被稳定拦截并跳转到中文拦截页。
3. 浏览器重启后，只要会话未到 `endAt`，专注限制仍然生效。
4. 用户可以通过带摩擦感的应急解锁流程进行临时放行或结束会话。
5. 用户可以查看本地专注统计，但扩展不上传云端数据。

### 2.3 非目标

1. 不解决用户手动禁用或卸载扩展后的绕过问题。
2. 不在首版支持企业强制安装与策略管控。
3. 不在首版支持路径级白名单规则。
4. 不在首版支持移动端 Edge。

## 3. 用户与场景

### 3.1 目标用户

1. 需要在浏览器中上课、查资料、写作业的学生。
2. 需要长时间在线学习，但容易被娱乐网站打断的自学用户。

### 3.2 典型场景

1. 用户准备学习前，输入本次目标并设置专注时长。
2. 用户学习中尝试打开非白名单网站，被立即拦截并看到剩余时间。
3. 用户遇到真实紧急情况，填写原因并完成答题后短时放行。
4. 用户学习结束后查看今日、本周专注统计和被拦截记录。

## 4. 全局产品规则

1. 一次只能存在一个进行中的专注会话。
2. 学习目的不能为空。
3. 未选择时长时不能开始专注。
4. 专注进行中不得修改白名单。
5. 专注进行中不得修改任何配置项。
6. 专注进行中不能直接结束限制，只能通过应急解锁流程结束。
7. 临时放行 10 分钟的作用范围为全部网站。
8. 开始专注后，已打开的非白名单标签页也要立即拦截并跳转到拦截页。
9. 冷却时间功能必须存在，但是否启用以及时长由用户在非专注状态下配置。
10. 自定义时长上限和冷却时间由用户自行配置，统计数据固定保留最近 7 天。
11. InPrivate 支持保留，但需用户在 Edge 中手动开启。
12. 开发和测试阶段需要提供快速退出按钮，该能力不默认暴露在正式版界面中。
13. 临时放行期间应在浏览器旁展示简洁倒计时提醒窗，倒计时结束后自动关闭并恢复拦截。

## 5. 用户可配置项

| 配置项 | 配置方式 | 修改时机 | 说明 |
| --- | --- | --- | --- |
| 自定义时长上限 | 预设范围选择 | 仅未开始专注时 | 不支持任意值输入 |
| 统计数据保留说明 | 固定提示 | 仅保留最近 7 天，超过一周自动清理 | 无需用户配置 |
| 冷却时间启用 | 开关 | 仅未开始专注时 | 控制答题失败后是否进入冷却 |
| 冷却时间时长 | 分钟数输入 | 仅未开始专注时 | 仅在启用冷却时生效 |
| 主题模式 | 黑/白主题切换 | 任意时机 | 不影响专注规则 |

## 6. 核心数据模型字段表

### 6.1 `focusSession`

| 字段 | 类型 | 必填 | 说明 | 规则 |
| --- | --- | --- | --- | --- |
| `id` | string | 是 | 会话唯一标识 | 建议使用时间戳加随机串 |
| `status` | enum | 是 | `idle` / `active` / `completed` / `unlocked` | 任一时刻仅一个 `active` |
| `purpose` | string | 是 | 本次学习目的 | 去首尾空格后不能为空 |
| `startAt` | number | 是 | 开始时间戳 | 毫秒级 |
| `endAt` | number | 是 | 结束时间戳 | 必须大于 `startAt` |
| `plannedMinutes` | number | 是 | 原始设定时长 | 正整数 |
| `remainingMinutesSnapshot` | number | 否 | 用于 UI 快速展示 | 可实时计算 |
| `whitelistSnapshot` | array | 是 | 开始时固化的白名单快照 | 专注期间以快照为准 |
| `allowAllUntil` | number | 否 | 临时放行截止时间 | 仅应急解锁后存在 |

## 8. 核心流程

### 8.1 开始专注

1. 用户打开弹窗并点击“开始专注”。
2. 输入学习目的，选择快捷时长或自定义时长。
3. 系统校验输入内容和上限规则。
4. 创建 `focusSession` 并保存白名单快照。
5. 写入动态拦截规则。
6. 立刻扫描现有标签页并处理非白名单页面。

### 8.2 正常拦截

1. 用户访问非白名单网站。
2. 动态规则将主框架导航重定向到拦截页。
3. 后台记录本次被拦截域名与时间。
4. 拦截页展示剩余时间、目的和应急解锁入口。

### 8.3 应急解锁

1. 用户进入解锁页并填写原因。
2. 系统生成 5 道两位数乘法题。
3. 用户一次性提交答案。
4. 若全部答对，展示“临时放行 10 分钟”与“结束本次专注”二选一。
5. 解锁成功后，页面展示本次解锁类型、结果提示，并在可恢复时提供“回到相关页面”按钮。
6. 若失败，记录失败结果；开启冷却时写入冷却截止时间。

### 8.4 会话结束

1. 到达 `endAt` 或用户通过解锁流程选择结束会话。
2. 后台清理动态拦截规则。
3. 会话状态变为 `completed` 或 `unlocked`。
4. 汇总数据写入统计记录。

## 9. 异常与边界条件

1. 浏览器重启后需要基于 `endAt` 恢复会话，不只依赖 `alarms`。
2. service worker 休眠时，关键状态不能只放内存。
3. 专注期间打开白名单站点不应被误拦截。
4. 非法域名输入需要提示并禁止保存。
5. 统计保留天数到期后需要自动清理旧记录。
6. InPrivate 未开启时，应给出说明提示而非误导用户认为已生效。
7. 开发态快速退出只用于调试，不能污染正式版行为。

## 10. MVP 验收标准

1. 不填写学习目的时不能开始专注。
2. 不选择时长时不能开始专注。
3. 专注中访问非白名单站点时进入拦截页。
4. 白名单站点能正常访问。
5. 浏览器重启后，未结束会话仍持续生效。
6. 专注进行中不能修改白名单和配置项。
7. 开始专注后，已打开的非白名单标签页会被立即处理。
8. 应急解锁必须先填写原因并答完 5 道题。
9. 答题通过后，用户可以在“临时放行 10 分钟”和“结束本次专注”中二选一。
10. 冷却时间功能存在，且其启用状态与时长可以配置。
11. 临时放行期间浏览器旁会显示倒计时提醒窗，结束后自动关闭并恢复拦截。
12. 解锁成功后会展示本次解锁类型，并在可恢复时提供返回相关页面按钮。
13. 数据汇总页能查看专注、拦截和解锁记录。
14. 所有正式 UI 默认使用中文。
